Section LOCALIZE

  • Use one of the available methods to define the localization and possibly to optimize it to a minimum or a maximum.
  • Section path: CP2K_INPUT / FORCE_EVAL / PROPERTIES / LINRES / LOCALIZE
  • This section cannot be repeated.

    • SECTION_PARAMETERS
    &LOCALIZE {Logical}
    controls the activation of the MOS localization procedure
    This keyword cannot be repeated and it expects precisely one logical.
    The lone keyword behaves as a switch to .TRUE.
    Default value: .FALSE.

    This keyword cannot be repeated and it expects precisely one logical.
    The lone keyword behaves as a switch to .TRUE.

    • ENERGY_RANGE
    ENERGY_RANGE {Real} {Real}
    Select the orbitals to be localized within the given energy range.This type of selection cannot be added on top of the selection through a LIST. It reads to reals that are lower and higher boundaries of the energy range.
    This keyword cannot be repeated and it expects precisely 2 reals.
    Default values: 0.00000000E+000 0.00000000E+000
    Default unit: [eV]

    • MAX_ITER

    • METHOD
    METHOD {Keyword}
    Method of optimization if any
    This keyword cannot be repeated and it expects precisely one keyword.
    Default value: JACOBI
    List of valid keywords:
    • CRAZY
      A new fast method is applied, might be slightly less robust than jacobi, but usually much faster
    • DIRECT
      Using a direct minimisation approacha
    • JACOBI
      Using 2 x 2 rotations of the orbitals, slow but robust
    • L1SD
      Steepest descent minimization of an approximate l1 norm
    • NONE
      No localization is applied
    • SCDM
      Use QR factorization
