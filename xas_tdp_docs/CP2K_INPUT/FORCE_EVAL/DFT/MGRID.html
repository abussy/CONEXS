Section MGRID

  • Keyword descriptions

    • COMMENSURATE
    COMMENSURATE {Logical}
    If the grids should be commensurate. If true overrides the progression factor and the cutoffs of the sub grids
    This keyword cannot be repeated and it expects precisely one logical.
    The lone keyword behaves as a switch to .TRUE.
    Default value: .FALSE.
    • CUTOFF
    CUTOFF {Real}
    The cutoff of the finest grid level. Default value for SE or DFTB calculation is 1.0 [Ry].
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 2.80000000E+002
    Default unit: [Ry]
    • MULTIGRID_CUTOFF
    MULTIGRID_CUTOFF {Real} ...
    List of cutoff values to set up multigrids manually
    This keyword cannot be repeated and it expects a list of reals.
    Alias names for this keyword: CUTOFF_LIST
    • MULTIGRID_SET
    MULTIGRID_SET {Logical}
    Activate a manual setting of the multigrids
    This keyword cannot be repeated and it expects precisely one logical.
    Default value: .FALSE.
    • NGRIDS
    NGRIDS {Integer}
    The number of multigrids to use
    This keyword cannot be repeated and it expects precisely one integer.
    Default value: 4
    • PROGRESSION_FACTOR
    PROGRESSION_FACTOR {Real}
    Factor used to find the cutoff of the multigrids that where not given explicitly
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 3.00000000E+000
    • REALSPACE
    REALSPACE {Logical}
    If both rho and rho_gspace are needed
    This keyword cannot be repeated and it expects precisely one logical.
    The lone keyword behaves as a switch to .TRUE.
    Default value: .FALSE.
    • REL_CUTOFF
    REL_CUTOFF {Real}
    Determines the grid at which a Gaussian is mapped, giving the cutoff used for a gaussian with alpha=1. A value 50+-10Ry might be required for highly accurate results, Or for simulations with a variable cell. Versions prior to 2.3 used a default of 30Ry.
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 4.00000000E+001
    Default unit: [Ry]
    Alias names for this keyword: RELATIVE_CUTOFF
    • SKIP_LOAD_BALANCE_DISTRIBUTED
    SKIP_LOAD_BALANCE_DISTRIBUTED {Logical}
    Skips load balancing on distributed multigrids. Memory usage is O(p) so may be used for all but the very largest runs.
    This keyword cannot be repeated and it expects precisely one logical.
    The lone keyword behaves as a switch to .TRUE.
    Default value: .FALSE.
