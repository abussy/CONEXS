Section OPTIMIZE_INPUT

  • describes an input optimization job, in which parameters in input files get optimized.
  • Section path: CP2K_INPUT / OPTIMIZE_INPUT
  • This section cannot be repeated.

  • Subsections

    • FORCE_MATCHING
    • HISTORY
    • RESTART
    • VARIABLE

  • Keywords

    • ACCURACY
    • ITER_START_VAL
    • MAX_FUN
    • METHOD
    • RANDOMIZE_VARIABLES
    • STEP_SIZE

  • Keyword descriptions

    • ACCURACY
    ACCURACY {Real}
    Final accuracy requested in optimization (RHOEND)
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 1.00000000E-005
    • ITER_START_VAL
    ITER_START_VAL {Integer}
    Used for restarting, starting value of the iteration
    This keyword cannot be repeated and it expects precisely one integer.
    Default value: 0
    • MAX_FUN
    MAX_FUN {Integer}
    Maximum number of function evaluations
    This keyword cannot be repeated and it expects precisely one integer.
    Default value: 5000
    • METHOD
    METHOD {Keyword}
    What kind of input optimization to perform.
    This keyword cannot be repeated and it expects precisely one keyword.
    Default value: FORCE_MATCHING
    List of valid keywords:
    • FORCE_MATCHING
      Perform a force matching minimization.
    • RANDOMIZE_VARIABLES
    RANDOMIZE_VARIABLES {Real}
    Percentage randomization of the free variables applied initially
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 0.00000000E+000
    • STEP_SIZE
    STEP_SIZE {Real}
    Initial step size for search algorithm (RHOBEG)
    This keyword cannot be repeated and it expects precisely one real.
    Default value: 5.00000000E-002
